## Prompt 提示词技巧：从模糊需求到可执行指令

Prompt的核心不是写得更长，而是**减少歧义、补足背景、明确交付方式，并让结果可以检查**。

一个实用的 Prompt 可以概括为：

> **角色 + 背景 + 任务 + 约束 + 输出格式 + 验收标准**

## 一、通用六段式结构

```text
角色：
你是一名具备……经验的专家。

背景：
当前情况是……目标用户是……最终用途是……

任务：
请完成……

要求：
1. 必须包含……
2. 不要……
3. 信息不足时……

输出格式：
使用表格、Markdown、JSON 或分步骤输出。

质量标准：
结果应准确、具体、可执行，并检查……
```

## 二、八个高频技巧

### 1. 用明确动作代替模糊表达

不要只说“分析一下”，而要明确模型需要完成的认知动作。

```text
较弱：帮我分析这篇文章。

更好：提取文章的核心观点、论据、研究方法和局限性，
并判断结论是否得到证据支持。
```

常用动作词包括：**提取、比较、分类、改写、评估、排序、计算、验证和生成**。

### 2. 提供必要背景

至少说明以下信息：

- 使用场景；
- 目标读者；
- 已有基础；
- 最终用途；
- 时间、预算或篇幅限制。

例如：

```text
面向没有机器学习基础的金融学本科生，
用课堂讲解语言解释，不使用复杂代码。
```

### 3. 明确输出格式

规定输出结构可以显著减少无效内容。

```text
请按以下格式输出：
1. 一句话结论
2. 三个关键发现
3. 风险清单
4. 可执行建议
5. 仍需确认的信息
```

如果结果需要交给程序处理，应要求输出严格的 JSON，并明确字段名称和数据类型。

### 4. 给出评价标准

“请认真完成”不是可执行标准。更有效的表达是：

```text
每条建议必须：
- 能在两周内执行；
- 说明预期收益；
- 说明所需资源；
- 指出主要风险。
```

评价标准告诉模型什么才算“高质量答案”。

### 5. 用示例校准风格

当格式、语气或分类方式要求较高时，提供一个正向示例，通常比使用大量抽象形容词更有效。

```text
参考风格：
输入：利润下降
输出：利润下降主要来自毛利率收缩，而非收入减少。

请按照这种“结论先行、解释随后”的方式处理其余内容。
```

### 6. 规定信息不足时的处理方式

为了减少模型编造，可以明确要求：

```text
如果材料中没有答案，请写“材料未提供”，不要推测。
如果必须作出假设，请单独列出假设及其影响。
```

当缺少的信息会显著改变结论时，可以要求模型先提出不超过三个澄清问题。

### 7. 将复杂任务拆成阶段

复杂问题可以拆解为：

1. 提取事实；
2. 区分事实、观点与推测；
3. 比较不同解释；
4. 形成结论；
5. 检查证据缺口。

应要求模型提供**简明的判断依据和可核查证据**，不必要求输出冗长的内部思维过程。

### 8. 增加自检机制

在 Prompt 末尾加入检查清单：

```text
输出前请检查：
- 是否遗漏任务要求；
- 数字和单位是否一致；
- 结论是否有材料支持；
- 是否混淆事实与推测；
- 格式是否符合要求。
```

## 三、不同任务需要补充什么

| 任务类型 | Prompt 中应补充的内容 |
| --- | --- |
| 写作 | 读者、目的、语气、篇幅和参考风格 |
| 总结 | 摘要长度、必须保留的信息、是否允许推断 |
| 数据分析 | 指标定义、时间范围、异常值规则和图表要求 |
| 编程 | 技术栈、运行环境、输入输出、边界情况和测试要求 |
| 学术研究 | 数据来源、证据等级、引用要求和研究局限 |
| 决策 | 备选方案、评价指标、权重和风险承受能力 |
| 内容生成 | 发布平台、目标人群、传播目的和禁用表达 |
| Agent 执行 | 可用工具、授权范围、完成条件和失败处理方式 |

## 四、常见误区

- 角色描述很长，但任务本身不清楚；
- 一次加入多个相互冲突的目标；
- 只要求“专业、深入、高质量”，却没有评价标准；
- 没有规定输出长度和结构；
- 缺少原始材料，却要求确定结论；
- 要求最新信息或精确引用，却不允许联网核实；
- 一味追求超长 Prompt，导致重点被稀释。

## 五、可直接复用的模板

```text
你是一名具备【专业领域】经验的专家。

背景：
【说明项目、材料、受众和使用场景】

任务：
请完成【具体任务】。

具体要求：
1. 【必须包含的内容】
2. 【分析维度】
3. 【限制条件】
4. 信息不足时，请标注缺失信息，不要编造。
5. 如果必须作出假设，请明确列出假设。

输出格式：
1. 核心结论
2. 关键分析
3. 风险或局限
4. 可执行建议
5. 待确认问题

质量标准：
- 结论先行；
- 内容具体；
- 依据清楚；
- 建议可执行；
- 避免空泛表述；
- 输出前进行一致性检查。
```

## 六、最终原则

高质量 Prompt 的本质，是把隐含需求转化为明确任务，把主观期待转化为可检查的验收标准。

写完提示词后，可以用三个问题快速检查：

1. 模型是否清楚要完成什么？
2. 模型是否知道结果应该长什么样？
3. 我是否能够判断结果合不合格？

如果三个问题都能回答“是”，这个 Prompt 通常已经具备较好的可执行性。
